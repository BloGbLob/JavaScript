1. Nazwa pętli: Pętla FOR

Konstrukcja:


for (inicjalizacja; warunek; modyfikacja) { // kod do wykonania }

Opis: Pętla używana wtedy, gdy z góry znamy dokładną liczbę powtórzeń. Ma wbudowany licznik (np. i++).

Schemat blokowy: Start -> Inicjalizacja licznika -> Sprawdzenie warunku -> (Jeśli TAK) Wykonanie kodu -> Modyfikacja licznika -> Powrót do warunku. (Jeśli NIE) Koniec pętli.

2. Nazwa pętli: Pętla WHILE (z kontrolowanym wejściem)

Konstrukcja:


while (warunek) { // kod do wykonania }

Opis: Warunek jest sprawdzany na samym początku (na wejściu). Jeśli warunek od początku jest fałszywy, pętla nie wykona się ani razu.

Schemat blokowy: Start -> Sprawdzenie warunku -> (Jeśli TAK) Wykonanie kodu -> Powrót do warunku. (Jeśli NIE) Koniec pętli.

3. Nazwa pętli: Pętla DO...WHILE (z kontrolowanym wyjściem)

Konstrukcja:


do { // kod do wykonania } while (warunek);

Opis: Warunek jest sprawdzany na samym końcu (na wyjściu). Oznacza to, że pętla wykona się zawsze co najmniej jeden raz, nawet если warunek jest fałszywy.

Schemat blokowy: Start -> Wykonanie kodu -> Sprawdzenie warunku -> (Jeśli TAK) Powrót do wykonania kodu. (Jeśli NIE) Koniec pętli.
